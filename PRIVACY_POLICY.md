# Privacy Policy (Draft)

## Data Collected
- Account identity data: name, email, auth provider
- Session metadata: IP, user-agent, session timestamps
- Event metadata created by hosts
- Participant wallet addresses, references, transaction signatures, deposit statuses

## Purpose
- Authenticate users
- Operate event deposit workflows
- Verify on-chain payments and maintain receipts

## Data Sharing
- On-chain transaction data is public by blockchain nature.
- No sale of personal data.

## Retention
- Keep auth/session and payment records as required for operations and dispute review.

## Security
- Password hashing, HttpOnly session cookies, backend authorization checks.

## User Rights
- Request account data review/deletion (subject to legal/financial record constraints).

Note: Review with legal counsel before production launch.
